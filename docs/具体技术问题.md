# 研究点二：应急场景下无人机网络资源分配系统 - 具体技术问题

## 简要概述（80字）

针对应急场景下无人机网络的资源分配问题，设计并实现基于Lyapunov优化理论的任务调度系统。系统需解决动态网络拓扑建模、任务粒度资源分配算法实现、多目标优化权衡、实时调度与性能监控等关键技术问题，并通过Web平台提供算法验证与可视化分析能力。

---

## 具体技术问题描述（500字以内）

面对应急场景下无人机网络的任务调度与资源分配问题，现有系统存在调度粒度粗、缺乏理论优化保障、实时性能监控不足等局限性，本系统需要解决网络拓扑建模、任务粒度调度算法实现、并发调度与状态管理、性能监控与分析等核心技术问题。在网络拓扑建模方面，系统需要构建动态的无人机网络拓扑结构，支持节点（网关、边缘节点、卫星节点）和链路的灵活配置，通过Floyd-Warshall算法计算节点间最短路径，构建邻接矩阵和路径矩阵，为后续任务调度提供路径计算基础，同时支持网络拓扑的可视化展示和实时更新，确保管理员能够直观地配置和调整网络结构。在任务粒度调度算法实现方面，系统需要将研究点一基于用户维度的算法细化为任务维度，实现更精细的资源分配策略，采用时隙离散化调度模型（1秒/时隙），为每个任务动态分配计算节点和传输路径，调度算法基于Lyapunov Drift-Plus-Penalty优化框架，通过成本函数Drift + V × Penalty平衡队列稳定性与系统性能，其中Drift衡量队列长度的平方和，Penalty为延迟、能耗、负载的加权组合，实现过程中采用贪心随机化策略，每个时隙进行多次随机迭代，根据队列长度优先级分配资源，并通过早停机制在成本稳定时终止迭代。在并发调度与状态管理方面，系统需要解决算法引擎与API服务的并发访问问题，调度器运行在独立goroutine中，通过ticker机制实现周期性调度，使用读写锁保护TaskManager、System等共享状态，防止竞态条件，同时区分持久化对象（Task）和临时计算上下文（TaskSnapshot），Task包含完整的历史指标（MetricsHistory），TaskSnapshot仅包含单次调度所需的队列、资源分配等临时数据，避免数据混淆。在性能监控与分析方面，系统需要提供多维度的性能指标采集和可视化功能，在每个时隙记录任务的传输延迟、计算延迟、传输能耗、计算能耗、队列长度、处理进度、资源分配比例等指标，形成完整的执行历史，前端通过图表展示延迟趋势、能耗趋势、队列变化等时序数据，并支持按时隙回溯查询，为算法性能评估和优化提供数据支撑，整个系统通过RESTful API和Web界面实现前后端分离，采用JWT认证保障安全性，使用SQLite轻量级数据库存储基础数据和历史指标。
